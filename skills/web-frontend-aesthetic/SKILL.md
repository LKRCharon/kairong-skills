# Web Frontend Aesthetic

## When to use
- Build a frontend page that should feel intentional rather than template-like.
- Improve a web UI that works functionally but lacks visual personality.
- Create a visual prototype where typography, spacing, color direction, and motion carry meaning.

## Workflow
1. Choose a visual direction first.
   Name the intended mood and make the implementation serve it consistently.
2. Avoid default choices unless the existing product language requires them.
   Pick typography, palette, spacing rhythm, and component shapes deliberately.
3. Design the page as a composition, not a stack of cards.
   Use contrast, asymmetry, and spacing rhythm to produce hierarchy.
4. Make motion purposeful.
   Favor a few meaningful transitions over many decorative micro-interactions.
5. Check both desktop and mobile.
   Responsive adaptation should preserve the concept, not merely shrink the layout.

## Quality bar
- The UI should not look like an anonymous AI-generated template.
- Typography and spacing should create clear hierarchy even before color is added.
- Accent colors should be limited and strategic.
- Interactions and states should feel coherent with the chosen visual direction.
- The design should stay legible, usable, and visually confident across viewport sizes.
