# Theme: Kairong Clean Blue

This theme is extracted from a polished one-page resume reference provided by the user.

Use it when the user wants a resume that feels:
- clean
- technical
- restrained
- systems-oriented
- recruiter-friendly within 30 seconds

## Core visual idea

The page uses a mostly monochrome base with a single blue accent color.
The blue is not decorative; it is structural.
It appears in:
- section headers
- horizontal divider rules
- selected inline technical labels
- small icon accents

Overall impression:
- crisp
- efficient
- calm
- competent

## Layout

- Single-column A4 layout
- Tight but breathable vertical rhythm
- Strong top identity block
- Section blocks separated by thin horizontal blue rules
- No cards, panels, shadows, or decorative containers inside the page

This layout is especially good for:
- CS internship resumes
- ML systems resumes
- infrastructure or inference resumes
- research resumes that still need to read like industry applications

## Typography

- Sans-serif throughout
- Name is bold, uppercase, and clearly larger than the rest of the page
- Subtitle is smaller and muted
- Section headers are uppercase, compact, and colored blue
- Body text is restrained, around 12-13 pt equivalent
- Titles are denser than academic CV styles and avoid excessive whitespace

## Color system

Approximate palette:
- Accent blue: around `#5B8DEF` to `#6FA4FF`
- Main text: dark slate / charcoal
- Secondary text: muted gray
- Background: white

Guidelines:
- Keep the blue light enough to feel modern but saturated enough to anchor the page
- Use dark text for all primary reading
- Reserve blue for structure and emphasis, not large paragraphs

## Icon style

- Small inline icons only
- Prefer outline icons
- Use them in contact rows, dates, or compact metadata
- Keep icon weight subtle so they support scanning without turning decorative

Avoid:
- large hero icons
- colorful icons
- icon badges or icon cards

## Content style

This theme works best when the content is:
- concrete
- metric-backed
- systems-aware
- easy to skim

Project formatting should usually follow this structure:
- project name in bold
- short technical subtitle or stack line in blue
- compact date or timeframe on the right
- 3-5 bullets with strong action and measurable results

## When to choose this theme

Prefer this theme for:
- software engineering internships
- ML infra
- inference optimization
- systems + LLM crossover roles
- research resumes that should still look industry-ready

It is less ideal when the user wants:
- a visually expressive research portfolio
- a design-heavy creative resume
- a long-form academic CV

## Implementation notes

When reproducing this theme:
- use one blue accent only
- keep horizontal rules thin
- avoid heavy borders
- keep the page visually flat and sharp
- let spacing and typography, not decoration, carry the hierarchy
