# Role Emphasis Guide

Use this guide when the same user is applying to different AI-related roles and the resume should change emphasis.

## One primary direction

Default to one main direction per resume:
- `LLM`
- `multimodal`
- `inference optimization`
- `AI4Science`

Do not present the user as a broad "everything in AI" profile unless the target role is genuinely broad.

## LLM roles

Highlight:
- model adaptation and post-training
- data construction
- evaluation
- ablation and error analysis
- papers or experiments that show method understanding

Useful terms:
- `LoRA`
- `QLoRA`
- `DPO`
- `SFT`
- `alignment`
- `RAG`
- `evaluation pipeline`

## Multimodal roles

Highlight:
- modality coverage such as vision, audio, or video
- encoder or fusion design
- datasets and annotation setup
- multimodal evaluation and failure cases

Useful terms:
- `vision-language`
- `cross-modal alignment`
- `video understanding`
- `audio-text`
- `instruction tuning`

## Inference optimization / infra roles

Highlight:
- serving architecture
- latency and throughput gains
- memory optimization
- deployment and scaling
- distributed or systems engineering

Useful terms:
- `vLLM`
- `FSDP`
- `DeepSpeed`
- `FlashAttention`
- `CUDA`
- `TensorRT-LLM`
- `KV cache`
- `throughput`
- `latency`

## AI4Science roles

Highlight:
- scientific problem framing
- domain datasets
- simulation, prediction, or scientific evaluation
- reproducibility and experiment design
- how the work connects to real scientific questions

Useful terms:
- `molecular property prediction`
- `scientific benchmark`
- `ablation`
- `reproducibility`
- `pipeline`

## How to rewrite emphasis

For model research roles:
- emphasize methods, experiments, ablations, and findings

For infra or inference roles:
- emphasize systems, optimization, deployment, and measurable performance gains

For multimodal roles:
- emphasize data modality, fusion strategy, and evaluation setup

For AI4Science roles:
- emphasize domain problem, scientific relevance, and rigorous experiment logic

## Red flags

Avoid lines like:
- "participated in large model training"
- "familiar with multimodal"
- "have good coding ability"

Prefer evidence-led wording:
- what you trained or optimized
- what stack you used
- what metric changed
- what failure analysis or ablation you personally handled
